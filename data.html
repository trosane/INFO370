Data & Analysis

What Did We Find?

Data Collection

Our data includes approximately 120,000 user comments, and 4500 timestamps of user submissions. This data was collected from the Reddit API using Python. Once the timestamps from each submission had been stored, the submissions were revisited, and every comment from the user who created the submission was fetched. A new comment object was created, which was another Python dictionary of username, subreddit, comment text, and the timestamp object of the comment. We used PowerShell to do our data set processing by writing a .ps1 script that could take in the raw data csv and return a new file containing a column with each UTC hour and a respective count of how many times posts were made during that hour.

Data Analysis

Our method for data analysis involved creating preliminary graphs of post frequency for cities in separate time zones. Then, the count of posts per hour were calculated and implemented to create a graph for each city. These graphs acted as our baseline for comparison when we created graphs for individual user post frequency. Our initial theory was that if a user whose post patterns coincided with that of a certain city's, they most likely lived in the same time zone of that city.

The graphs below show the usage data of one randomly chosen user from each city to depict how we can model a single user and visually match their pattern to a given city to determine where they might be.

We then tried to determine if different city graphs displayed variances that could be precipitated by obvious factors, such as sleeping and commute times. Below are three graphs of different cities in different time zones: Seattle, Shanghai, China and Tokyo. These graphs show posts during a day in each city.

Looking at these graphs, one can see that a common trend exists for cities worldwide. By acknowledging this, it can be said that it would be possible to compare a users individual posting graph with a city's time zone adjusted graph.
